# 毕业设计全过程管理系统需求规格说明书

## 1. 引言

### 1.1 项目背景
随着高等教育信息化的发展，传统的毕业设计管理模式已无法满足现代化教学管理的需求。为提高毕业设计管理效率，规范管理流程，实现全过程数字化管控，特开发此毕业设计全过程管理系统。

### 1.2 项目目标
构建一个集课题管理、过程指导、质量监控、审批流程于一体的综合性管理平台，实现毕业设计全过程的数字化、规范化管理。

### 1.3 适用范围
本系统适用于高等院校计算机相关专业毕业设计的全过程管理，包括课题申报、双选、指导、答辩等各阶段。

## 2. 系统概述

### 2.1 系统架构
采用前后端分离的微服务架构，后端使用Spring Boot + Spring Cloud，前端使用Vue.js，数据库采用MySQL + Redis。

### 2.2 技术栈
- 后端：Spring Boot 3.2.x, Spring Security 6.x, MyBatis Plus 3.5.x
- 消息队列：RabbitMQ 3.12.x（异步通知、文件处理任务）
- 搜索引擎：Elasticsearch 8.x（课题检索、论文全文检索）
- 前端：Vue 3.4.x, Ant Design Vue 4.x, ECharts 5.x（数据可视化）
- 数据库：MySQL 8.0.x, Redis 3.2.100
- 文件存储：MinIO（对象存储）
- 文档处理：OnlyOffice（在线编辑）、Apache POI（文档解析）
- 流程引擎：Flowable 7.x（轻量级工作流引擎）

- 基于RBAC（基于角色的访问控制）模型 
- 支持细粒度权限控制
- 实现数据权限隔离
- 支持权限继承和委托

## 4. 业务流程设计

### 4.1 四大阶段流程

#### 4.1.1 课题申报审查阶段
```
[开始] → 企业教师提交课题 → 高校教师预审 → 专业方向主管初审 → 督导教师终审 → 课题发布 → [结束]
```

#### 4.1.2 课题双选阶段
```
[开始] → 学生选报课题(最多3个) → 企业教师确认人选 → 双选结果确认 → 导师分配 → [结束]
```

#### 4.1.3 课题指导阶段
```
[开始] → 开题报告撰写 → 企业教师审查 → 开题任务书制定 → 项目指导记录 → 中期检查 → 论文指导 → [结束]
```

#### 4.1.4 毕设答辩阶段
```
[开始] → 答辩申请 → 论文提交 → 答辩资格审查 → [管理员统一分配交叉评阅] → 教师查看评阅任务 → 在线评阅（7天期限） → 提交评阅意见和评分 → 评分差异检测与处理 → 正式答辩 → 成绩评定 → 二次答辩(不合格者) → 文档打印 → [结束]
```

### 4.2 角色工作衔接逻辑

#### 4.2.1 课题审查流程
```
企业教师申报课题 → 高校教师预审 → 专业方向主管初审 → 督导教师终审 → 课题进入双选环节
```

#### 4.2.2 学生毕设流程
```
学生选报课题 → 企业教师确认人选 → 开题答辩(学生提交报告 + 企业教师审查) → 中期答辩(企业教师提交检查表 + 高校教师审查) → 课题指导(企业/高校教师分别指导项目/论文) → 论文提交 → 交叉评阅(管理员统一分配 + 教师查看任务 + 完成评阅) → 正式答辩(企业教师审查资格) → 成绩综合评定 → 文档打印
```

#### 4.2.3 不合格处理机制
```
任一环节审查不合格 → 失去正式答辩资格 → 仅能参加二次答辩 → 二次答辩通过后完成毕设文档打印

交叉评阅不合格处理：
交叉评阅评分不达标 → 进入修改完善环节 → 重新提交评阅 → 达标后进入正式答辩流程
```

#### 4.2.4 交叉评阅详细流程
```
[前置条件：学生完成论文提交]
↓
[管理员触发统一分配]
↓
[系统自动分配算法执行]
- 避嫌规则：指导教师不评阅自己指导的学生
- 领域匹配：根据课题类型和教师专业领域匹配
- 负载均衡：确保每位教师评阅数量均衡（默认上限可配置）
↓
[分配结果生成] → [管理员复核与手动调整（可选）] → [确认分配]
↓
[教师接收评阅任务通知]
↓
[教师登录系统查看评阅任务列表]
- 查看待评阅学生信息
- 下载论文文档
- 查看评分标准
↓
[开始在线评阅（7天期限）]
- 在线浏览论文
- 填写评阅意见
- 多维度评分（创新性、规范性、完整性等）
- 提交评阅结果
↓
[系统汇总评阅结果]
↓
[评分差异检测]
- 若评分差异>15分 → 专业方向主管介入仲裁
- 若正常 → 直接进入下一环节
↓
[学生查看评阅意见] → [根据意见修改（5天）] → [进入答辩流程]

异常处理分支：
- 超时未评阅 → 系统自动催办（提前1天、当天、超时1天三次） → 超时3天上报管理员介入
- 评阅质量问题 → 管理员人工复核 → 重新分配评阅
```

## 5. 功能需求详细说明

### 5.1 核心功能模块

#### 5.1.1 用户管理模块
- **用户注册与认证**
  - 统一身份认证
  - 多因素认证支持
  - 用户信息管理
  - 角色分配与权限配置

- **个人信息管理**
  - 基础信息维护
  - 密码修改
  - 联系方式管理
  - 简历信息维护

- **角色权限管理**
  - 各角色基础权限：个人中心、留言功能、下载专区、系统登录与退出
  - 督导教师：课题终审权限、审批结果修改权限、课题数量控制
  - 高校教师：课题预审权限、学生指导权限、成绩评定权限
  - 企业负责人：企业端统筹管理权限、师生信息管理权限
  - 专业方向主管：课题初审权限、指导记录审查权限
  - 企业教师：课题申报权限、学生指导权限、答辩资格审查权限
  - 学生：课题选报权限、文档提交权限、答辩参与权限

#### 5.1.2 课题管理模块

**5.1.2.1 课题申报功能**
- 课题基本信息录入
  - 课题名称、类型、来源
  - 研究内容、技术要求
  - 预期目标、创新点
  - 所需技能、工作量评估
  - 区分高职升本/3+1/实验班课题类型
- 课题文档附件上传
- 课题重复性检测
- 课题状态跟踪
- 课题修改记录跟踪

**5.1.2.2 四级审批流程**
- 企业教师初审
- 高校教师预审（可提交修改意见）
- 专业方向主管初审（分类型审查）
- 督导教师终审（最终把关）
- 审批意见记录与修改意见反馈
- 审批流程可视化跟踪
- 审批时效统计与逾期提醒
- 课题数量限制控制（**默认18个上限，支持管理员动态配置**）
- 流程退回机制（支持审批者退回并填写退回原因）

**5.1.2.3 课题双选管理**
- 学生选报管理（最多3个选题）
- 企业教师确认人选（每人最多16人）
- 双选匹配与调整机制
- 双选结果确认与公示
- 双选异常处理与重新匹配
- 企业负责人协同创新/校外课题教师指派
- 学生信息导出与统计

#### 5.1.3 过程管理模块

**5.1.3.1 文档管理功能**
- 支持文档类型：
  - 开题报告表
  - 开题任务书
  - 中期报告表
  - 指导记录表
  - 毕业论文
  - 项目代码文件
  - 答辩材料
  - 其他支撑材料

**文档处理功能**：
- **文档上传**
  - 支持多种格式（docx, pdf等）
  - 文件大小限制（单文件500MB）
  - 版本控制与历史记录
  - 上传进度显示与断点续传
  - 上传时间限制与逾期提醒

- **文档下载**
  - 批量下载支持
  - 下载权限控制（按角色分配）
  - 下载记录追踪
  - 断点续传支持

- **在线浏览**
  - 文档预览功能
  - 支持主流文档格式
  - 移动端适配
  - 浏览记录统计
  - 文档标注与批注功能

- **在线编辑**
  - 富文本编辑器
  - 协同编辑支持
  - 编辑历史记录
  - 自动保存功能
  - 修改意见标注功能

**5.1.3.2 答辩管理功能**
- 开题答辩安排与管理
- 中期答辩审查与结果记录
- 正式答辩资格审查
- 二次答辩安排与管理
- 答辩小组设置与成员分配
- 答辩结果录入与统计
- **交叉答辩评阅管理**
  - **管理员统一分配机制**
    - 支持一键自动分配
    - 分配前系统校验（资格审查通过、论文已提交）
    - 分配结果预览与手动调整
  - **避嫌分配算法**
    - 指导教师不评阅自己指导的学生
    - 同一研究组/实验室教师避嫌（可配置）
  - **领域匹配算法**
    - 根据课题类型匹配教师专业领域
    - 技术栈相似度计算（大数据、AI、Web开发等）
    - 权重设置：领域匹配60% + 负载均衁40%
  - **负载均衡算法**
    - 计算每位教师当前评阅数量
    - 确保每位教师评阅数量均衡（波动范围±2份）
    - 支持设置单教师评阅上限（默认10份，可配置）
  - **教师评阅功能**
    - 教师登录后查看待评阅任务列表
    - 在线查看学生论文（预览/下载）
    - 多维度评分表：创新性《规范性、完整性、技术难度、工作量
    - 在线填写评阅意见（富文本编辑器）
    - 提交评阅结果（评分+评语）
  - **评阅结果管理**
    - 评分差异检测：若两位评阅教师评分差异>15分，触发仲裁流程
    - 评阅质量分析：评语字数统计、评分分布分析
    - 评阅意见反馈：学生查看评阅意见并进行修改
    - 评阅结果统计：评分分布、通过率等
  - **超时提醒与催办**
    - 评阅期限7天，提前1天、当天、超时1天进行三次催办
    - 超时3天未完成自动上报管理员
    - 评阅进度实时可视化展示

**5.1.3.3 指导记录管理**
- 指导时间记录与统计
- 指导内容详细登记
- 指导效果评价与反馈
- 指导频次统计与分析
- 指导记录导出功能
- 专业方向主管审查功能
- 指导记录修改痕迹追踪

**5.1.3.4 进度跟踪**
- 阶段任务设置与时间节点控制
- 完成情况统计与进度分析
- 预警机制（逾期提醒、质量预警）
- 进度可视化展示（甘特图、进度条）
- 各环节时间节点设置与提醒
- 不合格处理流程跟踪

#### 5.1.4 审核评分模块

**5.1.4.1 多级审核机制**
- 四级课题审批流程（企业教师→高校教师→专业方向主管→督导教师）
- 各级审核任务分配与跟踪
- 在线审核功能与意见填写
- 审核结果汇总与统计分析
- 审核流程可视化追溯
- 审批时效监控与逾期提醒
- 审核结果修改权限控制

**5.1.4.2 评分管理**
- 评分标准配置与维护
- 多维度评分体系（开题、中期、论文、答辩等）
- 评分权重设置与调整
- **成绩组成规则配置**
  - 开题成绩占比（默认10%，可配置）
  - 中期成绩占比（默认10%，可配置）
  - 指导教师评分占比（默认40%，可配置）
  - 交叉评阅评分占比（默认20%，可配置）
  - 答辩成绩占比（默认20%，可配置）
  - 总成绩计算公式自动计算与验证
- 成绩计算与统计分析
- 成绩异议处理与复核
- 指导教师成绩录入与修改
- 学生成绩查询与打印权限控制

#### 5.1.5 审批流设计模块

**5.1.5.1 可视化审批流设计器**
- 拖拽式流程设计
- 节点类型支持：
  - 开始节点
  - 审批节点
  - 条件分支节点
  - 并行处理节点
  - 结束节点

**5.1.5.2 流程配置功能**
- 审批人指定方式
  - 固定人员
  - 角色指定
  - 动态选择
- 审批条件设置
- 超时处理机制
- 流程版本管理
- **流程退回与撤销机制**
  - 支持审批流程退回（带退回原因）
  - 支持学生主动撤回提交（限定时间窗口）
  - 支持管理员强制干预流程
  - 所有退回操作记录审计日志

**5.1.5.3 流程监控**
- 流程实例跟踪
- 审批时效分析
- 瓶颈节点识别
- 流程优化建议

#### 5.1.6 质量监控模块

**5.1.6.1 多维数据展现**
- **仪表盘展示**
  - 系统运行状态监控
  - 各阶段进度概览（课题申报、双选、指导、答辩）
  - 关键指标实时监控（课题通过率、指导完成率、答辩通过率）
  - 预警信息集中展示
  - 各角色工作量统计

- **统计图表**
  - 柱状图：各专业课题分布、各类型课题数量统计
  - 折线图：各阶段进度完成趋势、时间节点达成情况
  - 饼图：课题类型占比、各环节通过率分布
  - 雷达图：质量评估维度（课题质量、指导质量、文档质量）
  - 热力图：各时间段工作量分布

**5.1.6.2 质量指标体系**
- 课题质量评估（创新性、实用性、技术难度）
- 指导过程监控（指导频次、指导效果、及时性）
- 文档质量分析（完整性、规范性、创新性）
- 答辩质量统计（答辩通过率、成绩分布、评审意见）
- 各角色工作质量评估
- **增强数据统计分析**
  - 各教师指导质量对比分析（指导频次、学生通过率、平均成绩）
  - 课题类型与通过率关联分析（哪些类型课题通过率更高）
  - 历史数据趋势分析（年度对比、季度变化）
  - 学生毕设质量年度对比（成绩分布、优秀率对比）
  - 各专业方向毕设质量对比

**5.1.6.3 预警机制**
- **智能预警系统（基于规则引擎）**
  - **进度预警**
    - 提前3天黄色预警（邮件+站内信）
    - 提前1天橙色预警（邮件+站内信+短信）
    - 当天红色预警（全渠道推送）
    - 逾期紧急预警（上报管理员）
  - **质量预警**
    - 指导频次不足预警（数1周<1次）
    - 文档质量低于阈值预警（自动检测字数、格式）
    - 论文进度满预警（不同阶段完成度监控）
  - **答辩预警**
    - 论文未提交预警
    - 评阅不通过预警
    - 答辩资格审查不通过预警
  - **合规性预警**
    - 流程违规预警（跳过必要环节）
    - 权限异常预警（非法访问尝试）
  - **风险预警**
    - 不合格率过高预警（某教师/某专业>30%）
    - 关键节点延误预警（影响整体进度）
    - 某导师指导学生集体延误预警
- 自动预警通知与处理跟踪
- 预警规则可配置与动态调整

## 6. 非功能性需求

### 6.1 性能需求
- **系统响应时间：**
  - 普通页面加载 ≤ 1秒
  - 复杂统计报表 ≤ 3秒
  - 文件预览首屏 ≤ 2秒
  - API响应时间 ≤ 500ms
- **并发用户数：**支持1000+并发用户
- **文档处理能力：**
  - 单文件上传 ≤ 500MB
  - 批量下载 ≤ 2GB
  - 在线预览 ≤ 100MB
  - 支持断点续传
- **系统可用性：**99.9%以上（允许年度停机8.76小时）
- **数据库查询：**95%查询 ≤ 100ms
- **消息队列处理：**单条消息处理 ≤ 200ms

### 6.2 安全需求
- **传输层安全**
  - 数据传输加密（HTTPS + TLS 1.3）
  - WebSocket加密通信
- **认证与授权**
  - 用户身份认证（JWT + Refresh Token机制）
  - 双因子认证支持（短信验证码）
  - 单点登录（SSO）支持
- **数据安全**
  - 敏感数据加密存储（AES-256）
  - 密码存储（BCrypt哈希 + 盐值）
  - 数据备份保护（每日全量 + 实时增量）
  - 数据脱敏处理（日志中隐藏敏感信息）
- **权限控制**
  - RBAC基于角色的访问控制
  - 数据权限过滤（按角色隔离数据）
  - 接口权限校验（每个接口验证权限）
- **安全防护**
  - 防重放攻击（请求签名 + 时间戳验证）
  - SQL注入防护（参数化查询 + MyBatis Plus防注入）
  - XSS防护（前后端联防、输入过滤、输出转义）
  - CSRF防护（Token验证机制）
  - 文件上传安全（类型校验 + 文件内容检测 + 病毒扫描）
  - 接口限流（防止恶意请求，基于IP和用户级别限流）
- **并发控制**
  - 乐观锁机制（防止并发修改冲突）
  - 分布式锁（Redis实现资源互斥）
- **审计与监控**
  - 操作日志审计（所有关键操作记录）
  - 安全事件监控（异常登录、权限异常访问）
  - 实时告警机制（安全事件实时通知）
