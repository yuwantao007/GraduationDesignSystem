## 功能概述

学校管理模块用于管理3+1和实验班类型课题对应的学校信息。该模块仅对**系统管理员（SYSTEM_ADMIN）**开放。

### 功能清单

| 功能 | 权限码 | 说明 |
|------|--------|------|
| 学校列表 | `school:view` | 分页查询学校列表 |
| 创建学校 | `school:create` | 新建学校信息 |
| 编辑学校 | `school:edit` | 修改学校信息 |
| 删除学校 | `school:delete` | 删除学校记录 |
| 状态管理 | `school:status` | 启用/禁用学校 |

### 学校表结构

```sql
CREATE TABLE school_info (
    school_id VARCHAR(32) NOT NULL COMMENT '学校ID',
    school_name VARCHAR(100) NOT NULL COMMENT '学校名称',
    school_code VARCHAR(50) DEFAULT NULL COMMENT '学校编码',
    address VARCHAR(200) DEFAULT NULL COMMENT '详细地址',
    contact_person VARCHAR(50) DEFAULT NULL COMMENT '联系人',
    contact_phone VARCHAR(20) DEFAULT NULL COMMENT '联系电话',
    school_email VARCHAR(100) DEFAULT NULL COMMENT '学校邮箱',
    description VARCHAR(500) DEFAULT NULL COMMENT '学校简介',
    school_status TINYINT DEFAULT 1 COMMENT '状态（0-禁用 1-正常）',
    deleted TINYINT DEFAULT 0 COMMENT '逻辑删除（0-未删除 1-已删除）',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (school_id),
    UNIQUE KEY uk_school_name (school_name),
    UNIQUE KEY uk_school_code (school_code),
    KEY idx_school_status (school_status)
);
```

## 部署步骤

### 1. 后端部署

```powershell
# 进入后端目录
cd complete-backend

# 编译项目
mvn clean package -DskipTests

# 启动服务
java -jar target/complete-backend-0.0.1-SNAPSHOT.jar
```

### 2. 前端部署

```powershell

# 安装依赖
```

## 注意事项

1. **仅管理员可访问**：学校管理模块的所有接口和页面仅对系统管理员开放
2. **逻辑删除**：删除操作为逻辑删除，数据不会真正从数据库中移除
3. **唯一性约束**：学校名称和学校编码必须唯一
4. **状态管理**：禁用的学校不会出现在下拉选择列表中
