# 企业管理功能部署指南

## 📋 功能概述

企业管理模块是一个管理员专用功能，用于管理系统中的合作企业信息。包含以下功能：

- 企业列表查询（支持名称、编码、状态筛选）
- 企业信息查看（详细信息展示）
- 企业信息创建（含表单验证）
- 企业信息编辑（含表单验证）
- 企业信息删除（单个/批量删除）
- 企业状态管理（启用/禁用）

## 🔒 权限说明

**访问限制：** 仅系统管理员（SYSTEM_ADMIN）可见和操作

**权限列表：**
- `enterprise:view` - 查看企业列表和详情
- `enterprise:create` - 创建企业
- `enterprise:edit` - 编辑企业信息
- `enterprise:delete` - 删除企业
- `enterprise:status` - 管理企业状态

## 🚀 部署步骤

### 1. 执行SQL脚本（必须）

```bash
# 连接到MySQL数据库
mysql -u root -p

# 选择数据库
USE complete_system;

# 执行脚本
source /path/to/complete-backend/docs/sql/add_enterprise_permissions.sql;

# 或者在MySQL客户端中直接复制粘贴脚本内容执行
```

**验证权限是否添加成功：**
```sql
-- 查看企业管理权限
SELECT * FROM permission_info WHERE permission_id >= 500 AND permission_id < 520;

-- 查看系统管理员的企业权限
SELECT r.role_name, p.permission_code, p.permission_name
FROM role_permission rp
JOIN role_info r ON rp.role_id = r.role_id
JOIN permission_info p ON rp.permission_id = p.permission_id
WHERE r.role_code = 'SYSTEM_ADMIN' AND p.permission_id >= 500;
```

## 🐛 常见问题排查

### 问题1：菜单项不显示

**原因分析：**
1. 权限未正确配置到数据库
2. 用户未重新登录（旧token不包含新权限）
3. 前端权限检查条件不正确

**解决方案：**
```bash
# 1. 检查数据库权限配置
SELECT * FROM permission_info WHERE permission_code LIKE 'enterprise%';
SELECT * FROM role_permission WHERE permission_id >= 500;

# 2. 退出登录后重新登录（获取新的token和权限列表）

# 3. 检查浏览器控制台是否有报错
```

### 问题2：点击菜单报403错误

**原因分析：**
1. 后端权限注解配置错误
2. 前后端权限不一致
3. 用户权限列表未更新

**解决方案：**
```java
// 检查 EnterpriseController 是否有正确的权限注解
@PreAuthorize("hasRole('SYSTEM_ADMIN')")
public class EnterpriseController {
    // ...
}
```

### 问题3：表单验证不通过

**原因分析：**
1. 前后端验证规则不一致
2. 必填字段未填写
3. 格式不正确（手机号、邮箱）

**验证规则：**
- 企业名称：必填，最大100字符
- 企业编码：可选，最大50字符
- 联系人：可选，最大50字符
- 联系电话：可选，必须是11位手机号（1开头）
- 联系邮箱：可选，必须是有效邮箱格式
- 企业地址：可选，最大200字符
- 企业简介：可选，最大500字符

### 问题4：前端页面样式异常

**原因分析：**
1. Ant Design Vue组件未正确导入
2. 样式文件未加载
3. 浏览器缓存问题

**解决方案：**
