# 过程管理模块详细开发计划

## 模块总览

模块阶段：`TOPIC_GUIDANCE`（课题指导阶段，系统第三阶段）

模块范围：覆盖双选确认完成后，学生毕设执行的全过程：指导记录 → 文档管理 → 开题答辩 → 中期答辩 → 进度跟踪

涉及角色：学生、企业教师、高校教师、企业负责人

------

### 第一阶段：指导记录管理（优先级 🔴 最高）

#### 1.1 数据库设计

新增表：`guidance_record`（指导记录表）

CREATE TABLE `guidance_record` (

  `record_id`       VARCHAR(32)  NOT NULL COMMENT '指导记录ID',

  `student_id`      VARCHAR(32)  NOT NULL COMMENT '学生ID',

  `teacher_id`      VARCHAR(32)  NOT NULL COMMENT '指导教师ID',

  `topic_id`        VARCHAR(32)  NOT NULL COMMENT '课题ID',

  `guidance_type`   TINYINT      NOT NULL COMMENT '指导类型: 1=项目指导(企业教师), 2=论文指导(高校教师)',

  `guidance_date`   DATE         NOT NULL COMMENT '指导日期',

  `guidance_content` TEXT        NOT NULL COMMENT '指导内容（必填）',

  `document_id`     VARCHAR(32)  DEFAULT NULL COMMENT '关联文档ID（可选）',

  `is_deleted`      TINYINT      DEFAULT 0 COMMENT '逻辑删除: 0=正常, 1=删除',

  `create_time`     DATETIME     DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',

  `update_time`     DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

  PRIMARY KEY (`record_id`),

  KEY `idx_student_id` (`student_id`),

  KEY `idx_teacher_id` (`teacher_id`),

  KEY `idx_topic_id` (`topic_id`)

) COMMENT '指导记录表';

#### 1.2 后端实现清单

| 层次       | 文件                                                         | 说明                                  |
| :--------- | :----------------------------------------------------------- | :------------------------------------ |
| Entity     | `GuidanceRecord.java`                                        | 指导记录实体                          |

| DTO        | `CreateGuidanceDTO.java`                                     | 创建指导记录入参                      |
| VO         | `GuidanceRecordVO.java`                                      | 指导记录详情返回                      |

| Controller | `GuidanceRecordController.java`                              | 8个接口端点                           |

Controller 接口清单（8个）：

POST   /guidance/record                      新增指导记录（企业教师/高校教师）

DELETE /guidance/record/{recordId}           删除指导记录（本人可删）

GET    /guidance/record/my-students          我的学生列表（教师视角，含最新指导时间）

GET    /guidance/record/student/{studentId}  查看某学生的全部指导记录

GET    /guidance/record/my                   我的被指导记录（学生视角）

GET    /guidance/record/leader/list          指导记录总览（企业负责人，含筛选）

GET    /guidance/record/leader/export        导出指导记录 Excel（企业负责人）

GET    /guidance/record/{recordId}           指导记录详情

关键业务规则：

- 企业教师只能为 `topic_selection` 中已确认（状态=1）且课题归属于自己的学生提交指导记录
- 高校教师只能通过 `teacher_relationship` 配对关系为对应企业教师名下学生提交论文指导
- 只能删除自己创建的记录
- 指导内容必填，指导文档仅作关联（不在此表存储文件本身）

### 第二阶段：文档管理（优先级 🔴 高）

#### 2.1 数据库设计

新增表1：`document_info`（文档信息表）

CREATE TABLE `document_info` (

  `document_id`     VARCHAR(32)   NOT NULL COMMENT '文档ID',

  `student_id`      VARCHAR(32)   NOT NULL COMMENT '上传学生ID',

  `topic_id`        VARCHAR(32)   NOT NULL COMMENT '关联课题ID',

  `document_type`   TINYINT       NOT NULL COMMENT '文档类型: 1=项目代码, 2=论文文档, 3=开题报告, 4=中期检查表, 5=教师批注文档',

  `file_name`       VARCHAR(200)  NOT NULL COMMENT '原始文件名',

  `file_path`       VARCHAR(500)  NOT NULL COMMENT 'MinIO存储路径',

  `file_size`       BIGINT        NOT NULL COMMENT '文件大小(字节)',

  `file_suffix`     VARCHAR(20)   NOT NULL COMMENT '文件后缀(pdf/doc/docx/zip...)',

  `uploader_id`     VARCHAR(32)   NOT NULL COMMENT '上传人ID',

  `version`         INT           DEFAULT 1 COMMENT '版本号（同类型文档递增）',

  `is_latest`       TINYINT       DEFAULT 1 COMMENT '是否最新版本: 1=是, 0=否',

  `remark`          VARCHAR(500)  DEFAULT NULL COMMENT '备注（如：根据第X次指导意见修改）',

  `is_deleted`      TINYINT       DEFAULT 0 COMMENT '逻辑删除',

  `create_time`     DATETIME      DEFAULT CURRENT_TIMESTAMP,

  `update_time`     DATETIME      DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

  PRIMARY KEY (`document_id`),

  KEY `idx_student_topic` (`student_id`, `topic_id`),

  KEY `idx_doc_type` (`document_type`)

) COMMENT '文档信息表';

新增表2：`document_access_log`（文档访问日志表）

CREATE TABLE `document_access_log` (
